# Security Policy

## Supported Versions

Currently, only the latest release on the `master` branch is supported for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a core priority for Mimir, especially given its integration with financial brokerage APIs. 

If you discover a security vulnerability within Mimir (e.g., authentication bypasses, cross-site scripting, exposure of API keys, or timing attacks), please do not open a public issue.

Instead, responsibly disclose the vulnerability by sending an email to the project maintainers or by using GitHub's private vulnerability reporting feature (if enabled).

We will try to acknowledge your report within 48 hours and work with you to understand and patch the issue promptly.

### What to Include in Your Report
* A description of the vulnerability and its potential impact.
* Detailed steps to reproduce the issue (including whether you are running natively or via Docker).
* Any relevant logs or code snippets.

Thank you for helping keep Mimir secure!
